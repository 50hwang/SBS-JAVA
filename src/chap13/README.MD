# <제네릭스(Generics)>

> 2004년 Java 5가 제네릭스 개념을 추가했음.
> 제네릭스는 자바의 구조인 컬렉션 프레임워크에서 유용하게 사용됨.

## 제네릭스

> "매개변수화된 자료형"

- 클래스, 인터페이스 또는 메소드에 적용될 수 있는데, 클래스에 적용되면 제네릭 클래스, 인터페이스에 적용되면 제네릭 인터페이스, 메소드에 적용되면 제네릭 메소드라고 부름.

### 제네릭 메소드에 대해서 생각해보기

- 메소드에 인수를 넘길 때는 반드시 해당하는 매개변수의 자료형과 일치되는 자료형을 갖는 인수를 넘겨야 함.
    - 즉, 정수 매개변수에는 정수 인수를 넣어야 함.
- 그런데 제네릭스를 이용하면 매개변수에 다양한 자료형의 데이터를 넘길 수 있음.
    - 어떤 메소드가 void set(Integer)라고 한다면, 매개변수 x 자리에는 정수를 넣어야 함
    - 만약에, 여기에서 x 자리에 Integer 객체 뿐 아니라, 다른 타입의 객체를 인수로 넣고 싶다면, Integer 자리 자체를 매개변수로 취급하면 됨.
    - 제네릭스 개념이 바로 여기에서 시작됨.
    
```
void sort(Integer[] x)	// Integer 데이터 정렬
{ ... }
      ↓
void sort(T[] x)	// T에는 Integer 또는 Double이 올 수 있음.(또한 다른 클래스도 가능.)
{ ... }


```

    > 예제 코드 : 제네릭 클래스 Data, 코드 클래스 Code01
 
### 제네릭 클래스

- 이 클래스의 객체를 생성할 때에는 <> 기호를 사용하여 어떤 타입의 인수를 넣을지를 알려야 함.

```
클래스명<매개변수 타입 리스트> {
    ...
}
```

```
클래스명<타입> 객체명 = new 클래스명<타입>();
```
 
- 제네릭에 이용할 수 있는 자료형 : 레퍼런스 타입이어야 한다.
    - int, double 등과 같은 자바의 기본 자료형은 사용할 수 없다.
    - 만약 기본 자료형을 제네릭으로 사용하고자 한다면, (Code01 예제 코드처럼) wrapper 클래스인 Integer, Double 등을 이용해야 한다.
 
- 제네릭 타입도 엄격하게 문법에 따라야 한다.

```
Data<Integer> d1 = new Data<Integer>(100);
Data<String> d2 = new Data<String>("100");

if(d1 == d2) {	//제네릭 끼리의 비교에서 에러 발생!!
    System.out.println("Same data");
}
```

### 제한된 제네릭 타입

- <T extends V>
    - 제네릭 T 자리에는 클래스 타입이 V이거나 V 클래스의 하위 클래스 타입만 올 수 있다는 뜻.
